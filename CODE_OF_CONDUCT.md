# Code of Conduct

We want `cache-database` to be a project where people can ask basic questions,
share strong opinions, and still leave the discussion feeling respected.

## Our Standard

In this community, contributors and maintainers are expected to:

- act with respect, patience, and curiosity
- assume good intent before assuming carelessness
- give technical feedback directly without personal attacks
- keep discussions grounded in evidence, tradeoffs, and user impact
- make room for newer contributors and different experience levels

Unacceptable behavior includes:

- harassment, intimidation, or discrimination
- insulting or demeaning language
- repeated bad-faith arguing or deliberate disruption
- publishing private information without consent
- dismissing contributors instead of addressing the technical issue

## Scope

This applies to repository discussions, issues, pull requests, release notes,
and project spaces used to coordinate work.

## Reporting

For behavioral concerns, contact the maintainers through the private channel
configured for the repository. If a dedicated contact path is not yet listed,
open a maintainer-only moderation request through the repository administrators.

For security concerns, use the process in [SECURITY.md](SECURITY.md) instead of
filing a public issue.

## Enforcement

Maintainers may edit, hide, lock, or remove content that violates this code of
conduct. Repeated or serious violations can lead to temporary or permanent
removal from project spaces.

## Maintainer Commitment

We will enforce this code of conduct consistently, explain decisions when we
can, and prefer de-escalation over public conflict.
