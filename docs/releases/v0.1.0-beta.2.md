# v0.1.0-beta.2

## Highlights

This beta focuses on making CacheDB easier to evaluate from a real existing
PostgreSQL application, not only from a greenfield demo.

The main user-facing improvement is the Migration Planner: it can discover a
PostgreSQL schema, suggest root/child routes, generate scaffolds, warm a Redis
hot set in staging, compare PostgreSQL and CacheDB side by side, and produce a
readiness-oriented migration report.

## What Changed

### Added

- PostgreSQL schema discovery for planner-driven onboarding
- route candidate suggestions with apply-to-form support
- scaffold generation for entities, relation loaders, projections, and binding usage
- dry-run warm execution that does not mutate Redis
- staging warm execution for bounded hot windows
- side-by-side PostgreSQL vs CacheDB comparison
- migration readiness report with cutover action plan and rollback guidance
- admin UI navigation/category cleanup
- expanded English and Turkish onboarding docs

### Changed

- relation-heavy route guidance is now more explicit about projection/read-model requirements
- global sorted business views are steered toward ranked projections earlier
- demo docs now show the exact order of UI actions for seed, discover, plan, warm, compare, and report
- Spring Boot docs now clarify JDBC vs existing JPA `DataSource` setup

### Fixed

- projection-required comparison routes can resolve to the registered projection surface
- planner warm and comparison failures now provide clearer operational messages
- Turkish documentation in the main onboarding path has been rewritten for clearer, more natural wording

## Upgrade Notes

- Maven coordinates remain under `com.reactor.cachedb`.
- Use version `0.1.0-beta.2`.
- If you are testing migration from an existing ORM route, rebuild the app after adding generated entity/projection bindings before running warm or comparison.
- If comparison reports a mismatch, do not cut over even if CacheDB latency is lower. Fix parity and ordering first.

## Recommended Validation

Run:

```powershell
mvn -q -DskipTests package
```

For demo validation:

```powershell
./tools/ops/demo/run-spring-boot-load-demo.ps1
```

Then open:

- `http://127.0.0.1:8090/cachedb-admin`
- `http://127.0.0.1:8090/cachedb-admin/migration-planner`

## Status

This remains a public beta release. The project is suitable for evaluation,
staging pilots, and route-by-route migration rehearsal. Production adoption
should still go through warm-up, side-by-side comparison, rollback planning, and
route coverage review.
