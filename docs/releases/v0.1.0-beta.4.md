# v0.1.0-beta.4

## Highlights

This beta makes CacheDB's durable storage story provider-aware. PostgreSQL
remains the default public beta provider, while MSSQL is now available as an
explicit beta provider with live SQL Server evidence.

The release also fixes the public package contents so the JDBC and MSSQL storage
artifacts are included in the downloadable beta bundle.

## What Changed

### Added

- shared JDBC storage-provider layer for SQL dialect and write-behind support
- explicit PostgreSQL and MSSQL storage provider modules
- MSSQL write-behind flusher with version-guarded update/existence/insert semantics
- MSSQL outbox adapter and failure classifier
- live MSSQL provider evidence lane for write-behind, outbox checkpointing,
  migration SQL smoke, multi-pod apply-runner checkpoint locking, and SQL Server
  restart/reconnect behavior

### Changed

- user documentation now uses `selected SQL provider`, `source database`, and
  `durable SQL write-behind` language instead of implying PostgreSQL-only
  support
- PostgreSQL is documented as the default provider; MSSQL is documented as an
  explicit beta provider that must be wired deliberately
- release package generation now includes `cachedb-storage-jdbc` and
  `cachedb-storage-mssql`
- release publish tooling defaults to `main` and no longer creates a `codex/*`
  release branch by default

### Fixed

- root Maven metadata no longer describes CacheDB as PostgreSQL-backed only
- English and Turkish setup, migration, tuning, production recipe, provider SPI,
  and use-case docs now align with the provider-aware architecture

## Upgrade Notes

- Maven coordinates remain under `com.reactor.cachedb`.
- Use version `0.1.0-beta.4`.
- PostgreSQL users can keep the default starter wiring.
- MSSQL users must add `cachedb-storage-mssql`, provide the Microsoft SQL Server
  JDBC driver, and wire `MssqlWriteBehindFlusher` explicitly.
- Do not promote MSSQL to GA without staging SQL Server HA or Always On failover
  evidence, longer soak, and production-sized migration comparison.

## Recommended Validation

Run:

```powershell
mvn -q -DskipTests package
```

For production evidence:

```powershell
./tools/ci/run-production-evidence.ps1
./tools/ci/run-production-scenario-certification.ps1
./tools/ci/run-mssql-provider-evidence.ps1
```

## Status

This remains a public beta release. It is suitable for controlled production
pilots when Redis HA, route coverage, rollback, and side-by-side comparison are
validated for the application. GA still requires signed Maven Central publish,
staging Redis HA evidence, staging SQL Server HA evidence before MSSQL GA, and
full migration route coverage.
