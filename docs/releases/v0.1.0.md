# v0.1.0

Release date: 2026-06-14

This is the first non-beta CacheDB framework release.

## What Changed Since v0.1.0-beta.4

- The Maven project version is now `0.1.0`.
- The official distribution channel for this release is the GitHub Release
  asset `cache-database-0.1.0-github-release.zip`.
- Main onboarding docs now position CacheDB core and the default PostgreSQL
  provider path as stable, while keeping provider-specific limits explicit.
- Stable release packaging now uses a non-beta package label and stable release
  launch kit content.

## Stable Scope

- Redis-first entity repositories with explicit hot-set boundaries.
- Compile-time generated metadata and ORM-like APIs.
- PostgreSQL as the default durable SQL provider path.
- Bounded relation reads, projection/read-model recipes, and ranked projection
  guidance for heavy read routes.
- Migration Planner discovery, warm-up, side-by-side comparison, and cutover
  report flow.
- Multi-pod coordination, pod-unique consumers, Redis leader leases, and local
  Docker HA preflight tooling.
- Benchmark threshold, public API compatibility, production evidence, and GA
  readiness gates.

## Provider Boundaries

- PostgreSQL is the default stable provider path.
- MSSQL remains an explicitly selected beta provider. It has live SQL Server
  evidence and restart/reconnect checks, but this release does not claim SQL
  Server HA or Always On topology certification.
- Maven Central is optional for this release because GitHub Release is the
  selected official distribution channel.

## Production Cutover Boundary

This release can be used for production-oriented pilots and controlled
cutovers. A consuming application must still prove its own route coverage,
warm-up behavior, side-by-side comparison parity, Redis memory budget, rollback
plan, admin exposure model, and environment-specific HA behavior before cutting
production traffic over to CacheDB.

## Consumption

Use version `0.1.0`.

Download the official GitHub Release asset:

```text
cache-database-0.1.0-github-release.zip
```

Use the included Maven artifacts from the release bundle, or publish them to
your internal artifact repository.

## Evidence

Before announcing or consuming this release, verify:

- `Public Beta Readiness` is green for the release commit.
- `Production Evidence` is green for the release commit.
- `Production GA Release Readiness` is green for tag `v0.1.0`.
- The GitHub Release asset is attached and is not marked as prerelease.
