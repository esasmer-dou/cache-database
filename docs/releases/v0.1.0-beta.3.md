# v0.1.0-beta.3

## Highlights

This beta hardens CacheDB's production migration story for existing SQL
database systems. The focus is route-level contracts, tenant-safe Redis memory usage,
concrete outbox/CDC integration, and CI evidence that catches these behaviors
before release.

## What Changed

### Added

- production scenario certification lane with JSON and Markdown evidence
- concrete `PostgresOutboxExternalChangeFeedAdapter`
- route-level cache contracts with strict projection enforcement
- tenant quota support for hot-row and payload-level memory budgets
- composite hot policy support for count, time, state, and custom predicates
- migration warm checkpoint/resume support
- Redis memory calibration output for migration warm runs
- cache admission telemetry breakdowns

### Changed

- tenant memory budget now includes measured Redis entity payload bytes
- production evidence workflow now uses explicit Redis/source-database services
- benchmark threshold checks no longer fail on unstable fastest-shape noise in the read-shape microbenchmark
- production recipes and use-case docs now show route contracts, payload budget behavior, and outbox adapter usage

### Fixed

- repeated writes of the same entity no longer double-count tenant payload memory
- warm batch hydration now feeds tenant columns and payload estimates into route admission
- release bundle generation now copies the release note for the requested version

## Upgrade Notes

- Maven coordinates remain under `com.reactor.cachedb`.
- Use version `0.1.0-beta.3`.
- If the source database can be changed outside CacheDB, add an outbox/CDC feed before relying on Redis hot-set freshness.
- For relation-heavy or global sorted screens, define route contracts and keep projection-required routes in fail-fast mode for production.
- Re-run staging warm and side-by-side comparison after changing hot policy, tenant quota, or projection definitions.

## Recommended Validation

Run:

```powershell
mvn -q -DskipTests package
```

For production evidence:

```powershell
./tools/ci/run-production-evidence.ps1
./tools/ci/run-production-scenario-certification.ps1
```

## Status

This remains a public beta release. It is stronger for production pilots than
the previous beta, but GA still requires environment-specific staging evidence,
Redis HA validation, signed Maven Central publishing, and route coverage review.
