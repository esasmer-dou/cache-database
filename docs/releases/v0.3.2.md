# v0.3.2

Release tag: `v0.3.2`

CacheDB `0.3.2` closes the gap between framework examples and production-safe
consumer behavior. The core adds exact `BigDecimal` mapping support. Both REST
samples now enforce explicit command acceptance, active-set, relation, warm,
archive cursor, readiness and referential-integrity contracts.

## Highlights

- Exact monetary entity fields through generated codecs and JDBC writes.
- Route-specific admission policies instead of one generic sample policy.
- Bounded `IN (...)` relation loading instead of N+1 parent queries.
- Asynchronous warm jobs with one worker, bounded queue, status polling and `429` backpressure.
- `202 Accepted` writes, retryable `409` parent-not-durable responses and leaf-only sample deletes.
- Stable archive cursor using `(order_date, order_id)`.
- Real Redis 8 plus PostgreSQL and SQL Server integration evidence.
- Provider-parity CI and load scripts that wait for warm completion.

## Consumer version

```xml
<properties>
    <cachedb.version>0.3.2</cachedb.version>
</properties>
```

The release does not remove application-level production obligations. Every
consumer must still prove its route inventory, Redis memory budget, warm
coverage, side-by-side data parity, rollback path and environment-specific HA.
