# v0.2.0

Release etiketi: `v0.2.0`

## Ne Değişti?

- Framework ve örnek projeler `0.2.0` sürümüne taşındı.
- Annotation processor, Java 21 ile daha temiz çalışacak şekilde güncellendi.
- SQL Server için JDBC source loader güçlendirildi; `ORDER BY` içinde aynı id kolonu iki kez üretilmiyor.
- JDBC write-behind sırasında nullable SQL kolonları artık `NullPointerException` üretmeden taşınıyor.
- PostgreSQL ve MSSQL örnekleri, generated binding’leri `registerJdbcBacked(...)` ile kayıt edecek şekilde güncellendi.
- Warm/backfill akışı, elle yazılmış `JdbcTemplate` mantığı yerine `CacheWarmPlan` ve kayıtlı `EntityQueryLoader` üzerinden çalışıyor.
- Örnek projelere açık `ReadThroughConfig` eklendi.
- Hot timeline route’ları için çalıştırılabilir PowerShell yük testi script’leri eklendi.
- Sample şemaları write-behind için gerekli CacheDB runtime kolonlarıyla güncellendi.
- İngilizce ve Türkçe README’lere dependency, warm, read-through ve load-test örnekleri eklendi.

## Doğrulanan Örnek Kanıtı

Aşağıdaki sonuçlar Docker Desktop üzerinde, `0.2.0` paketi local Maven repository’ye kurulduktan sonra örnek projelerle alınmıştır.

| Örnek | Route profili | Komut şekli | Sonuç |
|---|---|---|---|
| PostgreSQL sample | `hot-timeline` | 10 müşteri, müşteri başına 20 sipariş, 10 müşteri penceresi warm, concurrency 4, 10 saniye | `ok=605`, `fail=0`, `p95=184 ms` |
| MSSQL sample | `hot-timeline` | 5 müşteri, müşteri başına 10 sipariş, 5 müşteri penceresi warm, concurrency 4, 10 saniye | `ok=6606`, `fail=0`, `p95=11.4 ms` |

Bu sonuçlar production benchmark değildir; örnek proje için smoke/load gate
kanıtıdır. Production kararı için staging veri hacmiyle SQL gecikmesi, Redis
bellek kullanımı, projection gecikmesi, write-behind backlog ve JVM GC birlikte
ölçülmelidir.

## Önerilen Dependency

```xml
<properties>
    <cachedb.version>0.2.0</cachedb.version>
</properties>

<dependency>
    <groupId>com.reactor.cachedb</groupId>
    <artifactId>cachedb-spring-boot-starter</artifactId>
    <version>${cachedb.version}</version>
</dependency>
```

MSSQL kullanıyorsan ayrıca şunu ekle:

```xml
<dependency>
    <groupId>com.reactor.cachedb</groupId>
    <artifactId>cachedb-storage-mssql</artifactId>
    <version>${cachedb.version}</version>
</dependency>
```

## Ürün Konumlandırması

CacheDB `0.2.0`, bilinçli tasarlanmış aktif route’lar için Redis-first
persistence/read-model katmanıdır. Her dinamik SQL sorgusunu otomatik hızlandıran
şeffaf bir middleware değildir.

BEST:

- Önce hızlı olması gereken route’u seç.
- O route için hot entity policy veya projection sözleşmesini tanımla.
- Mevcut SQL kayıtları Redis’e alınacaksa generated binding’i `registerJdbcBacked(...)` ile kayıt et.
- Trafiği taşımadan önce warm/backfill çalıştır.
- Arşiv, export ve geçmiş okumalarını açık SQL route’u olarak bırak.
- Cutover öncesi latency, veri eşleşmesi, Redis bellek kullanımı, projection gecikmesi ve write-behind backlog değerlerini ölç.

ANTI-PATTERN:

- Her tabloyu entity yapıp bütün sistemi aynı anda Redis’e taşımak.
- Uygulama açılınca tüm veritabanının Redis’e yükleneceğini varsaymak.
- Geniş entity sorgularının SQL’e düşüp sınırsız veriyi Redis’e doldurmasına izin vermek.
- Liste ekranlarında projection/read-model gerekirken full aggregate okumak.

## v0.1.0’dan Geçiş Notları

- Maven dependency’lerini `0.2.0` yap.
- Route mevcut SQL kayıtlarına ihtiyaç duyuyorsa `registerJdbcBacked(...)` ve açık warm planı kullan.
- Kontrollü miss durumlarında SQL’den okuyup Redis’i hydrate edeceksen `ReadThroughConfig` içinde `maxQueryLoadRows` değerini sıkı tut.
- MSSQL kullanıyorsan route-shaped warm sorgularını yeniden test et; SQL Server aynı `ORDER BY` kolonunun iki kez üretilmesini kabul etmez.
- Soft-delete kolonların nullable ise write-behind upsert yolu için bu sürüme geçmeden production’a çıkma.
